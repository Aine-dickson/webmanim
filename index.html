02

What the DSL looks like

Each object owns its physics, shape and event responses. The scene places instances and wires them together. The engine runs it at 60 fps.

pendulum.wm v0.1 
// physics, shape and events live together in the object
object Bob {
  shape: circle  r: 12  color: #ff6e9c

  state {
    theta:  1.2      omega: 0.0
    L:      160      pivot: [380, 60]
  }

  evolve method: rk4 {
    alpha  = -(9.8 / L) * sin(theta)
    omega += (alpha - 0.01 * omega) * dt
    theta += omega * dt
  }

  on drag    { theta = atan2(mouse[0] - pivot[0], mouse[1] - pivot[1]) }
  on release { omega = 0.0 }

  render { trail  color: #ff6e9c  opacity: 0.3 }
}

// the scene places instances and exposes controls
scene "Pendulum" {
  let bob = Bob

  control {
    slider bob.L     range: 60..240   label: "Length"
    slider bob.mass  range: 0.5..5.0  label: "Mass"
    button "Release" { bob.omega = 0.0 }
  }
}

What comes next

current JS engine. Full grammar, ODE scenes, spring / N-body, drag events, canvas renderer.
Phase 2 Rust parser. pest.rs PEG grammar, JSON AST, Language Server Protocol for editor tooling.
Phase 3 Rust sim core. State in SharedArrayBuffer, collision detection, full RK4 / Verlet / leapfrog.
Phase 4 SIMD kernels. Laplacian / gradient for PDE scenes. Heat diffusion, wave equation, fluid.
Phase 5 WebGL + 3-D. Large particle systems, 3-D coordinate space, orbital mechanics.
